## 1. 项目背景

### 1.1 项目概述

TravelAI 是一个基于人工智能的旅游推荐与规划系统，旨在帮助用户通过自然语言交互和智能算法，快速生成个性化的旅行行程方案。系统集成了 DeepSeek 大语言模型，能够根据用户的目的地、预算、天数等需求，自动生成包含每日行程安排、预算分配、景点推荐等内容的详细旅行计划。

### 1.2 业务痛点

| 痛点 | 描述 |
|------|------|
| 行程规划耗时 | 传统旅行规划需要查阅大量攻略，耗时且信息分散 |
| 预算难以控制 | 缺乏系统化的预算分配工具，容易超支 |
| 个性化不足 | 现有旅游平台推荐千篇一律，缺乏针对性 |
| 行程管理分散 | 行程信息散落在各处，难以统一管理和分享 |

### 1.3 项目目标

- 提供基于 AI 的智能行程生成服务，30 秒内生成完整旅行方案
- 支持行程的全生命周期管理（创建、编辑、状态跟踪、分享）
- 提供自然语言对话式 AI 助手，解答旅游相关问题
- 实现用户友好的 Web 界面，支持响应式布局

## 2. 用户角色与权限

### 2.1 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| 游客 | `GUEST` | 未登录用户，仅可访问首页、登录、注册页面 |
| 普通用户 | `USER` | 注册登录后，可使用全部功能 |
| 管理员 | `ADMIN` | 拥有系统管理权限，可管理所有用户和行程数据 |

### 2.2 权限矩阵

| 功能模块 | 游客 | 普通用户 | 管理员 |
|---------|------|---------|--------|
| 首页浏览 | ✅ | ✅ | ✅ |
| 用户注册/登录 | ✅ | — | — |
| AI 生成行程 | ❌ | ✅ | ✅ |
| 行程管理（CRUD） | ❌ | ✅（仅自己的） | ✅（全部） |
| 行程分享 | ❌ | ✅ | ✅ |
| AI 助手对话 | ❌ | ✅ | ✅ |
| 个人信息管理 | ❌ | ✅ | ✅ |

### 2.3 认证机制

- 采用 JWT（JSON Web Token）进行身份认证
- Token 有效期：2 小时（7200000 毫秒）
- Token 存储方式：前端 localStorage（`travelai_token`）
- 请求头格式：`Authorization: Bearer <token>`

## 3. 功能需求

### 3.1 用户管理模块

#### 3.1.1 用户注册（FR-USER-001）

- **输入**：用户名（2-20字符）、邮箱（合法格式）、密码（≥6位，含大小写字母和数字）
- **验证规则**：
  - 用户名唯一
  - 邮箱格式合法且唯一
  - 密码强度检测（需包含大小写字母和数字）
  - 确认密码一致性
  - 服务条款同意
- **输出**：注册成功返回用户信息，密码使用 BCrypt 加密存储

#### 3.1.2 用户登录（FR-USER-002）

- **输入**：用户名、密码
- **验证**：校验用户名密码匹配，账户状态为启用
- **输出**：返回 JWT Token 和用户信息
- **异常处理**：账户不存在/密码错误/账户禁用

#### 3.1.3 个人信息管理（FR-USER-003）

- 查看个人信息（用户名、邮箱、手机、头像等）
- 修改个人信息（昵称、手机、性别、生日、头像）
- 修改密码（需验证旧密码）

### 3.2 AI 行程推荐模块

#### 3.2.1 AI 生成行程（FR-RECOMMEND-001）

- **输入参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| destination | String | 是 | 目的地城市 |
| budget | Double | 是 | 预算金额（元） |
| days | Integer | 是 | 旅行天数 |
| startDate | String | 是 | 出发日期（YYYY-MM-DD） |
| travelers | Integer | 否 | 出行人数（默认1） |
| preferences | List\<String\> | 否 | 偏好标签（如"自然风光"、"美食"） |
| specialRequirements | String | 否 | 特殊需求 |

- **输出**：JSON 格式的完整行程方案，包含：
  - 每日行程安排（上午/下午/晚上的景点、时长、门票、交通、介绍）
  - 预算分配明细（住宿/餐饮/交通/门票/其他）
  - 旅行提示和注意事项
- **AI 模型**：DeepSeek API（deepseek-v4-flash）
- **降级策略**：API 不可用时返回预设的模拟行程数据

#### 3.2.2 行程优化（FR-RECOMMEND-002）

- 基于已有行程，根据用户反馈进行优化调整
- 支持局部修改（如更换某天的景点）

### 3.3 行程管理模块

#### 3.3.1 创建行程（FR-TRIP-001）

- 将 AI 生成的行程保存到数据库
- 存储内容包括：行程基本信息、每日行程、活动详情、预算明细
- 使用事务确保数据一致性

#### 3.3.2 查询行程列表（FR-TRIP-002）

- 分页查询当前用户的行程列表
- 支持按目的地/名称关键词搜索
- 支持按创建时间/出发时间排序
- 返回分页元数据（总条数、总页数、当前页）

#### 3.3.3 查看行程详情（FR-TRIP-003）

- 展示行程完整信息，包括每日活动和预算明细
- 展示行程状态标签

#### 3.3.4 修改行程（FR-TRIP-004）

- 编辑行程基本信息和活动详情
- 支持版本控制（乐观锁）

#### 3.3.5 删除行程（FR-TRIP-005）

- 软删除或硬删除行程
- 级联删除关联的每日行程、活动、预算明细

#### 3.3.6 行程状态管理（FR-TRIP-006）

- 状态定义：

| 状态码 | 状态文本 | 说明 |
|--------|---------|------|
| 0 | DRAFT | 草稿 |
| 1 | ACTIVE | 进行中 |
| 2 | COMPLETED | 已完成 |
| 3 | CANCELLED | 已取消 |

- 支持通过下拉菜单快速切换状态

#### 3.3.7 行程分享（FR-TRIP-007）

- 生成唯一分享 Token
- 通过分享链接查看行程详情
- 支持公开/私密切换

### 3.4 AI 助手对话模块

#### 3.4.1 创建对话（FR-CHAT-001）

- 创建新的对话会话
- 自动生成对话标题

#### 3.4.2 发送消息（FR-CHAT-002）

- 用户发送消息，AI 返回回复
- AI 回复基于 DeepSeek 大模型
- 支持上下文对话（保存对话历史）
- 降级策略：API 不可用时返回预设回复

#### 3.4.3 对话历史管理（FR-CHAT-003）

- 查看对话列表
- 查看单个对话的所有消息
- 删除对话

### 3.5 首页展示模块

#### 3.5.1 首页内容（FR-HOME-001）

- 展示系统介绍和核心功能入口
- 展示最近行程（登录后显示，最多4条）
- 快捷入口：AI 生成行程、AI 助手对话、行程管理

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | ≤ 3 秒（首屏） |
| API 响应时间 | ≤ 500ms（非 AI 接口） |
| AI 接口响应时间 | ≤ 30 秒（含超时处理） |
| 并发用户数 | ≥ 100 |
| 数据库查询 | ≤ 200ms（单表） |

### 4.2 安全需求

| 项目 | 要求 |
|------|------|
| 密码存储 | BCrypt 加密，禁止明文存储 |
| API 认证 | JWT Token，有效期2小时 |
| SQL 注入防护 | 使用 JPA 参数化查询 |
| XSS 防护 | 前端输入过滤，后端输出编码 |
| CORS 配置 | 配置允许的源、方法、头信息 |
| API 密钥保护 | 密钥存储在项目外部 .env 文件中，禁止硬编码 |

### 4.3 可用性需求

| 项目 | 要求 |
|------|------|
| 系统可用性 | ≥ 99% |
| 错误提示 | 友好的中文错误提示 |
| 降级策略 | AI 服务不可用时提供模拟数据 |
| 数据备份 | 数据库定期备份 |

### 4.4 兼容性需求

| 分辨率 | ≥ 1280×720（桌面端优先） |

- 统一的异常处理和响应格式
- 完善的日志记录（SLF4J + Logback）

## 5. 业务流程

### 5.1 用户注册登录流程

```
用户访问首页
  ├── 未登录 → 点击"登录"
  │     ├── 已有账号 → 输入用户名密码 → 验证通过 → 获取Token → 进入系统
  │     └── 无账号 → 点击"注册" → 填写信息 → 注册成功 → 自动登录 → 进入系统
  └── 已登录 → 直接使用系统功能
```

### 5.2 AI 行程生成与保存流程

```
用户进入AI推荐页面
  → 填写目的地、预算、天数、出发日期等信息
  → 点击"AI生成行程"
  → 系统调用DeepSeek API生成行程方案
  → 前端展示生成的行程详情（每日安排、预算分配、提示）
  → 用户点击"保存行程"
  → 系统将行程数据保存到数据库（trips + trip_days + trip_activities + trip_budget）
  → 提示保存成功
  → 用户可在"行程管理"页面查看已保存的行程
```

### 5.3 行程管理流程

```
用户进入行程管理页面
  → 系统分页加载用户行程列表
  → 用户可执行操作：
      ├── 查看详情 → 跳转行程详情页
      ├── 编辑行程 → 跳转编辑页
      ├── 修改状态 → 下拉选择状态（草稿/进行中/已完成/已取消）
      ├── 分享行程 → 生成分享链接 → 复制到剪贴板
      ├── 搜索行程 → 输入关键词搜索
      ├── 排序 → 按创建时间/出发时间排序
      └── 删除行程 → 确认删除 → 级联删除关联数据
```

### 5.4 AI 对话流程

```
用户进入AI助手页面
  → 系统加载历史对话列表
  → 用户选择"新建对话"或点击已有对话
  → 用户输入消息并发送
  → 系统调用DeepSeek API获取AI回复
  → 保存用户消息和AI回复到数据库
  → 前端展示对话内容
  → 用户可继续对话或删除对话
```

- MySQL 和 Redis 服务正常运行
